# Tag–Content Mapping Guide

Use this document as a reference when adding tags to event nodes via
`/admin/content`. Each event should have **at minimum 5 tags**, and
**must include a year tag**.

---

## Tag Taxonomy

Create these terms in the **Tags** vocabulary at `/admin/structure/taxonomy/manage/tags/add` before assigning them to events.

### Year Tags
`2016` · `2017` · `2018` · `2019` · `2020` · `2021` · `2022` · `2023` · `2024` · `2025`

### Topic Tags
`ARIA` · `Automated Testing` · `Captions` · `Cognitive` · `Color / Contrast`
· `Custom Components` · `Data Visualization` · `Design` · `Documents`
· `ePub / eBooks` · `Error Handling` · `Focus Management`
· `Forms` · `Gaming` · `Images / SVG` · `JAWS` · `Keyboard Navigation`

## Tagging Guidelines

1. **Year tag is required.** Every event must have the year it occurred.
2. **Minimum 5 tags per event.** Use the table above as a baseline.
3. **Screen reader tool tags** (`JAWS`, `NVDA`, `VoiceOver`, `TalkBack`,
   `Narrator`) should be added whenever the event discusses that specific tool.
4. **Topic tags** should reflect all primary subjects covered in the talk, not
   just the title.
5. **Disability-type tags** (`Cognitive`, `Low Vision`, `Motor`,
   `Deaf / Hard of Hearing`) should be added whenever the talk directly
   addresses users with those disabilities.
6. Correct the event titles and dates in the table above to match the exact
   node titles and dates in the CMS — the list above is a starting reference
   based on public event history.
